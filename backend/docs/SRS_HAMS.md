# **Software Requirements Specification (SRS)						โครงการ: Hospital Asset & Maintenance System (HAMS)** **ระบบบริหารจัดการครุภัณฑ์และงานซ่อมบำรุงโรงพยาบาล**

## **1\. บทนำ (Introduction)**

### **1.1 วัตถุประสงค์ (Purpose)**

เอกสาร Software Requirements Specification (SRS) ฉบับนี้จัดทำขึ้นเพื่ออธิบายความต้องการของระบบ **Hospital Asset & Maintenance System (HAMS)** ซึ่งเป็นเว็บแอพระบบบริหารจัดการครุภัณฑ์และงานซ่อมบำรุงของโรงพยาบาล โดยเอกสารนี้ใช้เป็นเอกสารอ้างอิงหลักระหว่างผู้พัฒนา ผู้ใช้งาน และผู้มีส่วนได้ส่วนเสีย เพื่อให้มีความเข้าใจตรงกันเกี่ยวกับขอบเขต ความสามารถ และข้อจำกัดของระบบ รวมถึงใช้เป็นแนวทางในการพัฒนา ทดสอบ และนำระบบไปใช้งานจริง

### **1.2 ขอบเขตของระบบ (Scope)**

Hospital Asset & Maintenance System (HAMS) เป็นระบบเว็บแอพที่ออกแบบมาเพื่อใช้บริหารจัดการครุภัณฑ์และงานซ่อมบำรุงภายในโรงพยาบาลอย่างเป็นศูนย์กลาง ครอบคลุมตั้งแต่การขึ้นทะเบียนครุภัณฑ์ การระบุแผนกที่ครุภัณฑ์ประจำอยู่ การยืม–คืน การเบิกจ่ายอะไหล่ การแจ้งซ่อมและติดตามงานซ่อมบำรุง ไปจนถึงการรายงานข้อมูลเชิงบริหารสำหรับผู้จัดการและผู้บริหารโรงพยาบาล

### **1.3 คำจำกัดความและคำย่อ (Definitions & Acronyms)**

* **HAMS**: Hospital Asset & Maintenance System  
* **Asset**: ครุภัณฑ์หรือทรัพย์สินของโรงพยาบาล  
* **Department**: แผนกหรือหน่วยงานภายในโรงพยาบาล  
* **Maintenance**: งานซ่อมบำรุงและบำรุงรักษาครุภัณฑ์  
* **Spare Part**: อะไหล่ที่ใช้ในการซ่อมบำรุง

## **2\. ภาพรวมของระบบ (Overall Description)**

### **2.1 มุมมองของผลิตภัณฑ์ (Product Perspective)**

ระบบเป็นเว็บแอพแบบศูนย์กลาง (Centralized Web Application) เพื่อให้ทุกหน่วยงานในโรงพยาบาลสามารถเข้าถึงข้อมูลครุภัณฑ์ชุดเดียวกัน ลดความซ้ำซ้อนของข้อมูล และสามารถติดตามสถานะของครุภัณฑ์ได้แบบเรียลไทม์ตามสิทธิ์ที่ได้รับ

### **2.2 ระดับผู้ใช้งาน (User Classes)**

**เจ้าหน้าที่พัสดุ (Parcel Staff)**

* ลงทะเบียนและแก้ไขข้อมูลครุภัณฑ์  
* จัดการจัดซื้ออะไหล่  
* ตรวจสอบสถานะครุภัณฑ์ทั้งหมด

**เจ้าหน้าที่ศูนย์ครุภัณฑ์ (Asset Center Staff)**

* จัดการการยืม–คืน และการโอนย้ายครุภัณฑ์   
* ตรวจสอบครุภัณฑ์ในศูนย์ครุภัณฑ์   
* ตรวจสอบสถานะครุภัณฑ์ในศูนย์ครุภัณฑ์   
* จัดการสต็อกอะไหล่

**เจ้าหน้าที่หน่วยงาน (Department Staff)**

* ขอเบิกหรือยืมครุภัณฑ์  
* ตรวจสอบครุภัณฑ์ในหน่วยงาน  
* ตรวจสอบสถานะครุภัณฑ์ในหน่วยงาน

**ช่างซ่อมบำรุง (Maintenance Staff)**

* บันทึกข้อมูลการซ่อมบำรุง  
* ติดตามสถานะงานซ่อม

**ผู้จัดการ / ผู้บริหาร (Manager)**

* ดูรายงานและแดชบอร์ดภาพรวม  
* ใช้ข้อมูลเพื่อการตัดสินใจเชิงบริหาร

**ผู้ดูแลระบบ (Admin)**

* จัดการสิทธิ์ผู้ใช้งานในระบบ

## **3\. ความต้องการทางฟังก์ชันของระบบ (Functional Requirements of HAMS)**

### **3.1 โมดูลการจัดการผู้ใช้และสิทธิ์ (HAMS-User & Access Module) (User & Access Control)**

* **FN-ACC-01** ระบบต้องสามารถกำหนดบทบาทผู้ใช้ (Asset Officer, Staff, Maintenance, Manager)  
* **FN-ACC-02** ระบบต้องควบคุมสิทธิ์การเข้าถึงข้อมูลตามบทบาทผู้ใช้


### **3.2 โมดูลบริหารจัดการครุภัณฑ์ (HAMS-Asset Module) (Asset Management)**

* **FN-AST-01** ระบบต้องสามารถบันทึกข้อมูลครุภัณฑ์ใหม่ได้  
* **FN-AST-02** ระบบต้องสามารถแก้ไขข้อมูลครุภัณฑ์ได้ตามสิทธิ์  
* **FN-AST-03** ระบบต้องสามารถระบุประเภทครุภัณฑ์ได้  
* **FN-AST-04** ระบบต้องสามารถระบุแผนกหรือหน่วยงานภายในโรงพยาบาลที่ครุภัณฑ์ประจำอยู่  
* **FN-AST-05** ระบบต้องสามารถแสดงสถานะครุภัณฑ์ (ใช้งาน, ชำรุด, อยู่ระหว่างซ่อม, จำหน่าย)
* **FN-AST-06** ระบบต้อง**ไม่ลบข้อมูลครุภัณฑ์** — การเปลี่ยนสถานะเท่านั้นเป็นวิธี “ยกเลิก” ครุภัณฑ์
* **FN-AST-07** เมื่อครุภัณฑ์ถูกรายงานสูญหาย ระบบต้องบันทึกประวัติการสูญหายลงใน `asset_lost` โดยระบุวันที่พบว่าสูญหาย สถานที่ที่พบล่าสุด และรายละเอียด/สาเหตุ
* **FN-AST-08** ระบบต้องรองรับกระบวนการจำหน่าย  2 ขั้นตอน: รอจำหน่าย (บันทึกเหตุผล) → จำหน่ายแล้ว (บันทึกสาเหตุและหมายเหตุ)
* **FN-AST-09** ระบบต้องสามารถแสดงประวัติการสูญหายและประวัติการจำหน่ายของครุภัณฑ์แต่ละรายการได้

### **3.3 โมดูลยืม–คืนครุภัณฑ์ (HAMS-Borrow & Return Module) (Borrow & Return Management)**

* **FN-BOR-01** ระบบต้องสามารถบันทึกข้อมูลการยืมครุภัณฑ์ได้  
* **FN-BOR-02** ระบบต้องสามารถบันทึกวันที่และเวลาที่เริ่มยืมและคืนครุภัณฑ์ได้  
* **FN-BOR-03** ระบบต้องสามารถคำนวณระยะเวลาที่ครุภัณฑ์ถูกยืมไปใช้งานได้โดยอัตโนมัติ  
* **FN-BOR-04** ระบบต้องสามารถแสดงสถานะการยืมครุภัณฑ์ (ยืมอยู่ / คืนแล้ว ) สำหรับเจ้าหน้าที่พัสดุ เพื่อใช้ในการติดตามและบริหารจัดการ  
* **FN-BOR-05** ระบบต้องสามารถแสดงประวัติการยืม–คืนของครุภัณฑ์แต่ละรายการได้  
* **FN-BOR-06** ระบบต้องสามารถแสดงสถานะก่อนยืมครุภัณฑ์แต่ละรายการได้ (ว่าง/ไม่ว่าง)สำหรับเจ้าหน้าที่หน่วยงาน ก่อนทำรายการยืมครุภัณฑ์

### **3.4 โมดูลเบิกจ่ายอะไหล่ (HAMS-Spare Part Module) (Spare Parts Requisition)**

* **FN-SPR-01** ระบบต้องสามารถบันทึกข้อมูลอะไหล่คงคลังได้  
* **FN-SPR-02** ระบบต้องสามารถบันทึกการเบิกจ่ายอะไหล่สำหรับการซ่อมบำรุงได้  
* **FN-SPR-03** ระบบต้องสามารถตัดสต็อกอะไหล่อัตโนมัติเมื่อมีการเบิกจ่าย  
* **FN-SPR-04** ระบบต้องสามารถแสดงประวัติการเบิกจ่ายอะไหล่ย้อนหลังได้

### **3.5 โมดูลซ่อมบำรุง (HAMS-Maintenance Module) (Maintenance Management)**

* **FN-MTN-01** ระบบต้องสามารถบันทึกใบแจ้งซ่อมครุภัณฑ์ได้  
* **FN-MTN-02** ระบบต้องสามารถกำหนดสถานะงานซ่อม (รอซ่อม / กำลังซ่อม / ซ่อมเสร็จ)  
* **FN-MTN-03** ระบบต้องสามารถบันทึกผลการซ่อมและอะไหล่ที่ใช้ได้  
* **FN-MTN-04** ระบบต้องสามารถแสดงประวัติการซ่อมบำรุงย้อนหลังของครุภัณฑ์ได้

### **3.6 การตรวจนับครุภัณฑ์ (Asset Audit)**

* **FN-AUD-01** ระบบต้องรองรับการบันทึกผลการตรวจนับครุภัณฑ์  
* **FN-AUD-02** ระบบต้องสามารถเปรียบเทียบผลการตรวจนับกับข้อมูลในระบบได้

### **3.7 รายงานและแดชบอร์ด (Reports & Dashboard)**

* **FN-REP-01** ระบบต้องสามารถสร้างรายงานครุภัณฑ์ตามประเภท สถานะ และหน่วยงานได้  
* **FN-REP-02** ระบบต้องสามารถแสดงรายงานภาพรวมสำหรับผู้บริหาร

## **4\. กฎทางธุรกิจ (Business Rules)**

* **BR-01** ครุภัณฑ์ทุกชิ้นต้องมีรหัสประจำตัวที่ไม่ซ้ำกัน  
* **BR-02** การแก้ไขข้อมูลครุภัณฑ์ต้องถูกบันทึกประวัติการเปลี่ยนแปลง  
* **BR-03** การจำหน่ายครุภัณฑ์ต้องได้รับการอนุมัติจากผู้มีอำนาจ  
* **BR-04** การยืม-คืนครุภัณฑ์ต้องถูกบันทึกประวัติการใช้งาน
* **BR-05** ครุภัณฑ์**ไม่มีการลบข้อมูล** — การเปลี่ยน `asset_status_id` เป็นวิธีเดียวที่เป็นไปได้ในการ “ยกเลิก” ครุภัณฑ์
* **BR-06** ทุกครั้งที่มีการรายงานว่าครุภัณฑ์สูญหาย ต้องมีการบันทึกประวัติลงใน `asset_lost` พร้อมรายละเอียดครบถ้วน
* **BR-07** การจำหน่ายต้องผ่านขั้นตอน “รอจำหน่าย” ก่อนเสมอ — ไม่สามารถบันทึกการจำหน่ายสำเร็จโดยไม่ผ่านขั้นตอน pending

## **5\. ความต้องการที่ไม่ใช่ฟังก์ชัน (Non-Functional Requirements)**

* **NF-01 Performance:** ระบบต้องสามารถค้นหาข้อมูลครุภัณฑ์ได้ภายในเวลาอันเหมาะสม  
* **NF-02 Availability:** ระบบต้องพร้อมใช้งานในเวลาทำการของโรงพยาบาล  
* **NF-03 Security:** ระบบต้องควบคุมการเข้าถึงข้อมูลตามบทบาทผู้ใช้  
* **NF-04 Data Integrity:** ระบบต้องป้องกันข้อมูลสูญหายและลดความซ้ำซ้อน
