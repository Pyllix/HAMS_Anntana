| **Normal Flow: (ยืมครุภัณฑ์)** | ระบบแสดงรายการครุภัณฑ์ทั้งหมดในพื้นที่รับผิดชอบ พร้อมข้อมูลต่อไปนี้ รหัสครุภัณฑ์ ชื่อครุภัณฑ์ ประเภทครุภัณฑ์ สถานะปัจจุบัน (ใช้งาน / ชำรุด / อยู่ระหว่างซ่อม / จำหน่าย) หน่วยงาน/แผนกที่ครุภัณฑ์ประจำอยู่ ปุ่มดูรายละเอียด (View Detail) ช่องค้นหาครุภัณฑ์ (ด้านบนของรายการ) ผู้ใช้เลือกครุภัณฑ์ที่ต้องการตรวจสอบ โดยกดปุ่ม "ดูรายละเอียด" ระบบแสดงหน้าจอรายละเอียดครุภัณฑ์ ประกอบด้วย ข้อมูลทั่วไปครุภัณฑ์ (รหัส, ชื่อ, ประเภท, วันที่ขึ้นทะเบียน) สถานะปัจจุบันของครุภัณฑ์ หน่วยงาน/แผนกที่รับผิดชอบ ประวัติการยืม–คืน ประวัติการซ่อมบำรุง ผู้ใช้ตรวจสอบข้อมูลครุภัณฑ์และสามารถกรองข้อมูลตามสถานะหรือหน่วยงานได้ ระบบแสดงผลการค้นหา/กรองตามเงื่อนไขที่ผู้ใช้กำหนด   |  |  |

| **Description:** | บันทึกข้อมูลการแจ้งซ่อมครุภัณฑ์ที่ชำรุดหรือต้องการการบำรุงรักษา เพื่อแจ้งให้เจ้าหน้าที่หรือช่างผู้รับผิดชอบดำเนินการแก้ไข |  |  |
| **Trigger:** | เมื่อผู้ใช้พบครุภัณฑ์ชำรุด หรือถึงรอบการบำรุงรักษา และกดเมนู "แจ้งซ่อมครุภัณฑ์" |  |  |
| **Preconditions: (Input)** | 1\.   	ผู้ใช้งานต้องเข้าสู่ระบบเรียบร้อยแล้ว และครุภัณฑ์ที่ต้องการแจ้งซ่อมต้องมีข้อมูลอยู่ในระบบ |  |  |

| **Normal Flow: (แสดงรายชื่อผู้ใช้)** | **ผู้ดูแลระบบ** เลือกเมนู **"จัดการผู้ใช้"** จากแถบเมนูหลักของระบบ **ระบบ**แสดงรายการบัญชีผู้ใช้ทั้งหมดในรูปแบบตาราง ประกอบด้วย: **รหัสผู้ใช้ (USER ID):** รหัสอ้างอิงตัวตนพนักงาน **ชื่อ-นามสกุล/อีเมล:** ข้อมูลระบุตัวตนผู้ใช้งาน **ชื่อผู้ใช้งาน (Username):** บัญชีสำหรับใช้เข้าสู่ระบบ **รหัสผ่าน:** แสดงในรูปแบบสัญลักษณ์ (\*\*\*\*\*\*\*\*) เพื่อความปลอดภัย  **แผนก/หน่วยงาน (Department):** หน่วยงานต้นสังกัด **ระดับผู้ใช้งาน(Role):** สิทธิ์การใช้งาน (เช่น Asset Officer, Maintenance, Manager) **สถานะ (Status):** สถานะการเข้าใช้งาน (ใช้งานปกติ / ระงับการใช้งาน)  **ปุ่มดูรายละเอียด (ไอคอนตา):** เพื่อดูข้อมูลเชิงลึกรายบุคคล **ปุ่มแก้ไข (ไอคอนดินสอ):** เพื่อเปลี่ยนแปลงข้อมูลหรือสิทธิ์การเข้าถึง **ปุ่มลบ (ไอคอนถังขยะ):** เพื่อจัดการบัญชีที่ไม่ได้ใช้งานแล้ว **ระบบแสดงช่องค้นหา (Search):** สำหรับค้นหาด้วย ชื่อ,อีเมล หรือ รหัสผู้ใช้ **ระบบแสดงตัวกรอง (Filter):** สำหรับกรองข้อมูลตาม ระดับผู้ใช้งาน หรือ สถานะของผู้ใช้งาน **ระบบแสดงปุ่ม "+ เพิ่มผู้ใช้ใหม่**: สำหรับลงทะเบียนบัญชีผู้ใช้งานรายใหม่เข้าสู่ระบบ **ผู้ดูแลระบบ** สามารถเลือกทำรายการที่ต้องการ (ดูรายละเอียด, เพิ่ม, แก้ไข หรือ ลบ)     |  |  |
| **Alternative Flows:    \[Alternative Flow 1 – เพิ่มผู ้ใช้\] \[Alternative  Flow 2 – แก้ไขข้อมูลผู้ใช้ \] \[Alternative  Flow 3 – ลบผู้ใช้\]**  |  **1a. ในขั้นตอนที่ 4 ของ Normal Flow** ถ้าผู้ใช้กดปุ่ม “เพิ่มผู้ใช้ใหม่” **ระบบ** แสดงหน้าต่างหรือหน้าจอสำหรับกรอกข้อมูลบัญชีใหม่ **ผู้ดูแลระบบ** กรอกรหัสพนักงาน, ชื่อ-นามสกุล, แผนก, กำหนด Username/Password และเลือกบทบาท **ผู้ดูแลระบบ** กดปุ่ม “บันทึก” **ระบบ** ตรวจสอบความถูกต้องของข้อมูล (Validation) และบันทึกข้อมูลลงฐานข้อมูล **ระบบ** แสดงข้อความยืนยันการเพิ่มผู้ใช้สำเร็จ และ**กลับไปขั้นตอนที่ 2 ของ Normal Flow  1b. ในขั้นตอนที่ 6 ของ Normal Flow** ผู้ใช้กดปุ่ม “แก้ไข(ไอคอนดินสอ)”  **ระบบ** ดึงข้อมูลปัจจุบันของผู้ใช้งานรายนั้นขึ้นมาแสดงบนแบบฟอร์มแก้ไข ประกอบด้วย: ข้อมูลพื้นฐาน (รหัสพนักงาน, ชื่อ-นามสกุล, แผนก)  บทบาทหน้าที่ (Role) เช่น Asset Officer, Staff, Maintenance, หรือ Manager สถานะการใช้งาน (Active / Inactive) **ผู้ดูแลระบบ** ทำการเปลี่ยนแปลงข้อมูลตามความต้องการ เช่น การเปลี่ยนแผนก หรือการปรับระดับสิทธิ์การเข้าถึงข้อมูล  **ผู้ดูแลระบบ** ตรวจสอบความถูกต้องและกดปุ่ม **“บันทึก” ระบบ** ทำการบันทึกข้อมูลชุดใหม่ลงฐานข้อมูล และทำการบันทึกประวัติการเปลี่ยนแปลง (Audit Log) เพื่อเก็บข้อมูลว่าใครเป็นผู้แก้ไขและแก้ไขเมื่อ **ระบบ** แสดงข้อความแจ้งเตือน “อัปเดตข้อมูลผู้ใช้สำเร็จ” และ**กลับไปขั้นตอนที่ 2 ของ Normal Flow 1c. ในขั้นตอนที่ 6 ของ Normal Flow** ผู้ใช้กดปุ่ม “ลบ(ไอคอนถังขยะ)”  **ระบบ** แสดงหน้าต่างยืนยันการลบ (Confirmation Dialog) เพื่อป้องกันความผิดพลาด **ผู้ดูแลระบบ** กดยืนยันการทำรายการ **ระบบ** ทำการเปลี่ยนสถานะผู้ใช้งานเป็น "ระงับการใช้งาน" หรือลบข้อมูลออกจากระบบ **ระบบ** แจ้งเตือนการลบผู้ใช้สำเร็จและ**กลับไปขั้นตอนที่ 2 ของ Normal Flow** |  |  |
